## D

### DailyTimeIntervalSchedule

相關位置：

- `JobsController.CreateOrUpdateJob()`
- `JobsController.ApplyCronOrDailySchedule()`

## Disable / 停用

相關位置：

- `MainViewModel.DisableJobAsync()`
- `JobsController.PauseJob()`
- `JobsController.IsJobDisabledAsync()`

關鍵欄位：

- `IsDisabled`

防護：

- UI 停用執行按鈕。
- API 拒絕手動觸發。
- 停用狀態編輯儲存不掛 live trigger。

## E

### Enable / Resume / 啟用

相關位置：

- `MainViewModel.EnableJobAsync()`
- `JobsController.ResumeJob()`
- `JobsController.RebuildTriggersForResumeAsync()`
- `JobsController.BuildTrigger()`

重點：

- 不直接 `ResumeJob` 舊 trigger。
- 從 `OriginalTriggers` 重建下一個未來 trigger。
- 避免停用期間 misfire 補跑。

## EventId

相關位置：

- `ProcessRunnerJob.SaveLog()`
- `JobLogEntryViewModel.cs`
- `JobsController.GetAllJobs()`
- `JobsController.GetJobLogs()`

執行事件：

- `100` 工作已開始
- `107` 自動觸發
- `110` 手動觸發

- `201` 動作已完成
- `203` 動作失敗
- `322` 並發略過
- `323` 每週間隔略過
- `328` 動作已停止

管理事件：

- `106` 建立排程
- `140` 更新排程
- `141` 刪除排程
- `142` 停用排程
- `143` 啟用排程

## MisfireActionFireAndProceed / Misfire / 錯過盡快啟動

相關位置：

- `ScheduleRequest.cs`
- `AddJobViewModel.cs`
- `JobsController.CreateOrUpdateJob()`
- `JobsController.BuildTrigger()`

用途：

- 決定錯過排程是否盡快補一次。

防爆設計：

- 未勾選：計算下一個未來時間。
- 勾選：SimpleTrigger 會限流，避免補跑大量歷史觸發。
- 停用後啟用：不補跑停用期間。

## W

### WeeklyInterval

相關位置：

- `TriggerDto.cs`
- `EditTriggerViewModel.cs`
- `JobsController.CreateOrUpdateJob()`
- `ProcessRunnerJob.cs`

用途：

- 每隔 N 週執行。
